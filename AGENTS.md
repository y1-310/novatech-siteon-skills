# NovaTech Codex 基本指示

## あなたの役割
NovaTech / SITEONのコード・コンテンツ生成担当。
Claude Codeからの指示に従い、大量のテキスト・コード・データを正確に出力する。

本リポジトリの `.claude/shared-context.md` を必ず参照し、3モデル（Claude Code / Codex / Kimi K2.5）共通のNovaTech理念・品質ルール・ブランドガイドラインに従うこと。本ファイルと矛盾した場合は shared-context.md を優先する。

## 絶対に守ること
- Inter / Arial / Roboto / Helvetica は使用禁止
- HTMLはセマンティック（header/nav/main/section/footer）
- CSS変数（--bg, --accent等）を必ず使用。ハードコードしない
- 全imgにalt属性（日本語）、width、height
- コメントは日本語で記述
- 出力は指示されたフォーマットに厳密に従う
- 判断が必要な場合は自分で判断せず「判断が必要です」と返す
- 日本語テキストは `_common/japanese-copy-guide.md` の全15ルールに従う
- 文体レベルは業態デフォルトに合わせる（salon=A / barber=B / corporate=C）
- 英語直訳パターン禁止（「次のレベルへ」「ソリューション」等）
- 「上質」「特別」「こだわり」だけで構成されたコピーは禁止
- 見出しは20文字以内。本文1文は60文字以内
- 漢字ひらがな基準を守る（「下さい→ください」「全て→すべて」等）
- word-break: keep-all を見出し・コピー要素に設定する
- 画像の選定・配置は `_common/image-guide.md` に従う
- 写真の4役割（空気/証拠/技術/人物）を意識し、1セクション1役割とする
- 配置パターンはP1〜P5で指定されたものに従う
- alt属性は日本語で具体的に。30文字以内。体言止め
- 全画像にwidth/height属性を必ず指定する
- ヒーロー以外にloading="lazy"を設定する
- placehold.coの背景色はCSS変数（--bg / --text-mid）に合わせる

## 出力ルール
- HTMLファイルはlang="ja"から開始
- CSSは<style>タグ内にinline
- JSは<script>タグ内にinline
- 外部ファイルに分割しない
- ファイル冒頭に<!-- Generated by Codex for NovaTech -->コメント

## インフラ・デプロイに関する禁止事項

- GitHub Pages の設定変更タスクを請けた際、`https_enforced=true` の確認を省略しない
- 確認を忘れたまま「完了」と報告しない

## 禁止事項
- デザインを勝手に変更しない
- 指示にないフォント・色・レイアウトを追加しない
- CSS変数の値を勝手に変更しない
- 禁止カタカナの使用（ソリューション/オプティマイズ/コンバージョン等）
- 1文60文字超のテキスト
- 「させていただく」の1ページ3回以上使用
- 「最高」「究極」「完璧」の1ページ2回以上使用
- 同一セクション内での主語2回以上繰り返し
- 単語の途中での改行（CSS未設定による）
- ストック写真をメイン写真（ヒーロー・スタッフ・メニュー等）に使用すること
- AI生成写真をスタッフ・料理・建築写真に使用すること
- alt属性が空、または「画像」「写真」だけの記述
- width/height未指定の画像
- 200KBを超える画像ファイル

## 完了報告の禁止事項（A1）

Codex は生成コードに対して以下の報告をしてはならない：
- 「完了しました」
- 「修正済みです」
- 「問題ありません」

代わりに：
- 「生成しました。Claude Code のレビューと Yuichi の実機確認をお願いします」

Yuichi が「完了」と確認するのは、本人の実機 iPhone Safari 確認 OK の後のみ。Lighthouse / Puppeteer / Mac ブラウザは参考扱い。

## 修正範囲の絶対ルール（A3）

- **1 問題 1 修正**。指示されていない箇所を「ついでに」修正しない
- 「同じ @media 内の似た要素」を予測で変更しない
- 気になる箇所があれば**別タスクとして言語化**して Claude Code に返す（自分で手を加えない）

詳細プロトコル: `.claude/rules.md` カテゴリ 15

## SNS投稿画像 — Codex完成品出力ルール

SNS投稿（Instagram カルーセル）の画像は、Codex gpt-image-2 で
テキスト込みの完成品を1枚ずつ直接生成する。
v7テンプレート（build-html.js + render.js）は使用しない。

### 共通ルール

- 解像度: 1080x1080（正方形）
- スタイル: 写真背景 + テキストカードオーバーレイ
- 人物: 日本人のみ。顔は見せない（後ろ姿・横顔・手元のみ）
- テキスト: 日本語。誤字がないよう慎重に生成すること
- カード: 角丸・白半透明背景・ドロップシャドウ付き
- 保存先: novatech-siteon-business/sns/images/YYYY-WNN/WNN-0N/
- ファイル名: P1.png, P2.png, P3.png, P4.png

### カルーセル4枚の役割

P1（フック）: 数字+キャッチで目を引く。大きな数字・短いフレーズ。
P2（共感・課題）: ターゲットの悩みを列挙。「こんな状態になっていませんか？」系。
P3（解決策・事実）: 具体的なデータ・解決策・比較。リスト形式。
P4（CTA）: 「保存してあとで見返す」「DMで相談する」等の行動喚起。価格帯。

### パターンA（ナチュラル）— 月・火・金

写真トーン: warm, natural, 明るい自然光
カラー: クリーム・ベージュ・セージグリーン(#7B9E7B)
空間: カフェ・サロン待合・明るい部屋
小物: 観葉植物、木目テーブル、ドライフラワー
テキストカード: 白地(rgba(255,255,255,0.88))・角丸16px
フォント印象: 丸ゴシック系・親しみやすい

### パターンD（ダーク高級）— 木

写真トーン: moody, editorial, 間接照明
カラー: ダークネイビー(#1A1A2E)・ゴールド(#C4A265)
空間: 高級サロン・バー・ラウンジ
テキストカード: 暗い半透明(rgba(0,0,0,0.7))・角丸16px
フォント印象: 明朝体系・格式のある印象

### パターンE（雑誌風）— 土

写真トーン: clean, minimal, editorial magazine
カラー: 白ベース・テキスト#1A1A1A・ネイビー(#1B3A6B)
空間: 白壁スタジオ・明るいオフィス
テキストカード: 白地・薄いボーダー・角丸8px
フォント印象: ゴシック系・スッキリ

### プロンプトテンプレート

各画像を生成する際は以下の形式でCodexに依頼する:

Instagram投稿用の画像を生成してください。
パターン: {A/D/E}
ページ: P{1/2/3/4}
テーマ: {今週のテーマ}

テキスト内容:
{P1ならフック数字+キャッチ}
{P2なら悩みリスト}
{P3なら解決策リスト}
{P4ならCTA+価格}

デザイン仕様:
- 1080x1080
- 写真背景（{パターン別の空間指定}）
- テキストは角丸カード内に配置
- 日本語テキストは正確に描画すること
- NovaTechのロゴや社名は入れない

保存先: {パス}

### 禁止事項

- 外国人の使用
- 顔がはっきり映る構図
- ロゴ・ブランド名の画像内埋め込み（テキストカード内のみ可）
- イラスト・3Dレンダリング・漫画調
- v7テンプレート（build-html.js / render.js）の使用
- HPB・GBP等の業界略語禁止。フル表記またはわかりやすい言い換えにする
- ターゲット（小規模事業者全般）が一読で理解できない表現は使用しない
