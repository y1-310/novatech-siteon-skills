## 1. NovaTech / SITEON 理念

NovaTech は受託制作、SITEON はサブスク型 HP 制作サービス。両事業とも「Web 知識ゼロのオーナーが、AI を使えば月数千円でプロ品質のサイトを持てる世界」を目指す。

- **顧客像**：地方の小規模サロン（美容室・バーバー・ネイル・エステ）から開始し、飲食店・建築・会社 LP へ展開
- **価値提案**：従来の制作会社（数十万〜）ではなく、月額 9,800〜25,000 円のサブスクで保守・更新まで含めて提供
- **競争優位**：AI 3モデル体制（Claude Code / Codex / Kimi K2.5）による圧倒的な制作コスト削減とスピード

## 2. 役割分担（3モデル体制）

```
Claude Code (Opus 4.6, Max $100/月) ─── 司令塔（判断・計画・監督・チェック）
  ├── Codex (GPT-5.4, Plus $20/月) ─── コード生成 + 感性的な日本語（質重視）
  └── Kimi K2.5 API (従量 $5〜15/月) ─── 実務タスク全般（量重視・正確性）
```

| モデル | 担当 |
|--------|------|
| **Claude Code** | 計画・振り分け・最終承認 / 出力チェック / DM テンプレート設計 / SNS 戦略・企画 / git 操作・API 呼び出し |
| **Codex** | HTML/CSS/JS サイト生成 / コンセプト文・キャッチコピー / Instagram 投稿テンプレート HTML（初回のみ） |
| **Kimi K2.5** | 営業リスト生成・ふるい分け（Agent Swarm） / サロンカルテ整形 / 営業 DM 生成（テンプレ + パーソナライズ） / SNS 投稿文（X・Instagram） / 競合分析・比較資料（Visual Coding） / レポート・報告書 / メール文面（定型系） / データ整形・リスト処理 / デモサイトの視覚チェック（マルチモーダル） / クライアント写真の品質判定（マルチモーダル） / Google フォーム回答→JSON 変換 |

**振り分け原則**：「量が多く正確性重視」=Kimi、「量が少なく質重視」=Codex、「判断・最終承認」=Claude Code。Claude Code が 1 行でもコード・文章を自分で書こうとしたら、それは Codex か Kimi K2.5 に任せるべき作業。

## 3. 品質ルール要約（rules.md の要点）

### 禁止事項（最重要）
- **禁止フォント**：Inter / Arial / Roboto / Helvetica
- **アニメーション**：top / left / margin の使用禁止。`transform` と `opacity` のみ
- **!important**：noscript 内を除き使用禁止
- **日本語フォント全ウェイト読み込み禁止**（最大 3 ウェイト：300 / 400 / 500）
- **CSS/JS の外部ファイル分離禁止**（スタンダード時のみ。プロ以上は許可）
- **CSS 変数の値のハードコード禁止**
- **ビビッドな色の使用禁止**
- **デザインの勝手な変更禁止**（指示にないフォント・色・レイアウトの追加禁止）

### 必須事項
- **CSS 変数統一**：`--bg / --bg-alt / --bg-dark / --text / --text-mid / --text-light / --accent / --line / --r / --mw`
- **セマンティック HTML**：header / nav / main / section / footer
- **alt 属性**（日本語） / width / height / `loading="lazy"`（ヒーロー除く）
- **`lang="ja"`** を html 要素に。英語要素には `lang="en"` を個別指定
- **ファイル冒頭コメント**：`<!-- Generated by Codex for NovaTech -->`
- **セクション間 padding**：Desktop **120px** / Tablet **80px** / Mobile **72px**
- **セクション区切り線**：1px solid var(--line)、max-width: var(--mw)
- **セクションナンバリング統一**：01, 02, 03...
- **マーキー**：テキスト 2 回繰り返し、25s linear infinite、ディスプレイフォント italic
- **税込表記統一**
- **tel: リンク**を電話番号に設定
- **noscript 対応**：フェードイン→常時表示 / ナビ→フル表示 / マーキー→静止 / タブ→全展開
- **prefers-reduced-motion 対応**

### アクセシビリティ
- スキップナビ / aria-expanded / aria-label / focus-visible / フォーカストラップ
- コントラスト比 4.5:1 以上（WCAG 2.1 AA 準拠）
- タップターゲット 44px 以上
- 色だけに依存しない情報伝達

### 予約ボタン 5 箇所
ヘッダーナビ内 / ヒーロー内 / メニューセクション下 / フッター CTA / モバイルスティッキー（body に padding-bottom 追加必須）

### SEO
- title パターン：`[サロン名]｜[エリア名] [最寄り駅]の[業態]｜[特徴キーワード]`
- meta description（120〜160 文字）
- OGP 4 種 + twitter:card
- JSON-LD（業態別 @type：HairSalon / BarberShop / NailSalon / BeautySalon / Organization）に openingHoursSpecification / geo / aggregateRating / hasMap を含める
- FAQ 設置時は FAQPage を併記
- canonical URL（マルチページ時必須）
- viewport 設定

### パフォーマンス
- Lighthouse 全項目 90 点以上を目標
- Google Fonts に preconnect / font-display: swap / fallback スタック必須

詳細は `.claude/rules.md`（8 カテゴリ・約 50 項目）を参照。

## 4. ブランドガイドライン

### トーン

| 対象 | トーン |
|------|--------|
| **NovaTech / SITEON 公式（自社サイト・SNS）** | 親しみやすく、専門用語を避ける。「Web 知識ゼロでも分かる」言葉選び |
| **女性向けサロン LP（lp-salon, lp-nail-esthe）** | 詩的・優しい・繊細。「〜いたします」「〜のお手伝いを」 |
| **メンズ・バーバー LP（lp-barber）** | 力強い・断言的。「〜する」「〜だ」 |
| **コーポレート / グループ（lp-salon-group, lp-corporate）** | 信頼感・誠実。敬語ベース |
| **建築・住宅（lp-architect）** | 静謐・ミニマル。語りすぎない |
| **飲食店（lp-restaurant）** | 食欲をそそる・季節感 |

### 禁止表現
- 「絶対」「必ず」「100%」などの断定（景品表示法・薬機法対策）
- 「業界 No.1」「最安値」など根拠のない優良誤認表現
- 効果効能の断定（特に美容・エステ系）
- 競合他社・他サロンの誹謗中傷
- 個人を特定できる写真・口コミの無断使用

### 数値表記
- 料金は **税込** で統一
- 電話番号は `03-1234-5678` 形式（NAP 一貫性のため、サイト・GBP・SNS で完全一致）
- 営業時間は `10:00 - 19:00` 形式（半角ハイフン + 半角スペース）
- 日付は和暦不使用、`2026-04-08` または `2026 年 4 月 8 日`

### 画像の扱い
- クライアント提供写真を最優先
- 不足時のフォールバック順：
  1. placehold.co プレースホルダー → 写真揃い次第差し替え
  2. Unsplash / Pexels → 「実際のサロンではない」旨明記
  3. AI 画像生成（Gemini 等） → 内装イメージのみ。スタイル写真・スタッフ写真は不可

## 6. デモサイト生成ルール

- `hearing.json` の店舗名は業態に合うものをランダムで作成する
- 架空の住所・電話番号・スタッフ名も自動生成する
- 空白・プレースホルダーテキストのままにしない
- 画像は placehold.co または Unsplash / Pexels のフリー素材を使用する
- SNS 投稿用デモは「※デモサイトです」を投稿キャプション末尾に記載する

詳細ルール（業態別の名前例・料金帯・禁止事項など）は `skills/salon-interview/SKILL.md` の「デモサイト生成時の自動データ生成ルール」を参照。

## 日本語コピー品質ルール

全テキスト出力は `_common/japanese-copy-guide.md` に従う。

### 最重要ルール（全モデル共通）
1. 文体レベル（A/B/C）を業態に合わせて統一する
2. 英語直訳パターンは禁止（「次のレベルへ」「ソリューション」等）
3. 1文60文字以内。見出し20文字以内
4. 「上質」「特別」「こだわり」だけのコピーは禁止（具体性テスト）
5. 漢字ひらがな基準を守る（「下さい→ください」「全て→すべて」等）
6. word-break: keep-all を見出し・コピーに設定（改行制御）

### カタカナ禁止語（全業態共通）
ソリューション / オプティマイズ / コンバージョン / エンゲージメント /
プラットフォーム / アジャイル / イノベーション / シナジー /
エクスペリエンス / パーソナライズ / エンパワー / ジャーニー

### 詳細は _common/japanese-copy-guide.md を参照
