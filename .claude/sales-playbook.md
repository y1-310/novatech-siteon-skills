## 営業フェーズ詳細フロー（8段階・4部署制対応）

**フェーズ1：リスト生成**（営業・リサーチ部 = Kimi K2.5）
Yuichi「○○エリアの美容室でリスト作って」→ Google Maps検索 → CSV出力
収集項目：サロン名・住所・電話番号・業態・HP有無/URL・HPB/SNS URL・口コミ評価/件数

**フェーズ2：ふるい分け**（営業・リサーチ部）

- 第1フィルター（対象外を除外）：大手チェーン / プロ品質HP保有 / SNSアカウント皆無
- 第2フィルター（ランク付け）：
  - ★★★ Aランク：HP無し＋口コミ4.0以上/30件以上＋Instagram活発
  - ★★☆ Bランク：HP古い＋口コミ4.0以上
  - ★☆☆ Cランク：HPBのみ＋口コミ普通

**フェーズ3：事前調査**（営業・リサーチ部）
Aランク各サロンについて：Instagram・HPB・口コミの深掘り → 競合比較資料（近隣3件のHP状況）→ サロンカルテ作成

**フェーズ4：初回アプローチ**（営業・リサーチ部作成 → Yuichi送信）
サロンカルテからパーソナライズDM作成 → Yuichi確認 → Instagram DM送信 → 結果記録

**フェーズ5：デモサイト作成**（制作・品質部 = Codex）
返信ありのサロンのみ。Yuichi「○○のデモ作って」→ スキルでデモ生成（30分）→ デモURLをクライアントに送付

**フェーズ6：クロージング**（営業・リサーチ部作成 → Yuichi対話）
反応パターン別対応：
- 「いくら？」→ プラン比較資料
- 「ここ変えたい」→ 即修正
- 「検討します」→ 3日後フォロー
- 「高い」→ 再アプローチリスト

**フェーズ7：再アプローチ**（営業・リサーチ部自動管理）
初回反応なし：2ヶ月後→4ヶ月後→6ヶ月後の3回まで。別切り口のDMを自動生成→Yuichiに通知。3回反応なし→除外。

**フェーズ8：成約後処理**（営業・リサーチ部自動）
契約情報記録 / 紹介プログラムタイマーセット（1ヶ月後）/ ポートフォリオ更新マーク / 制作・品質部へ引き渡し

## 顧客ステータス遷移

```
未接触（prospect）→ DM送信済（contacted）→ 商談中（negotiating）
→ デモ提示済（demo_sent）→ 提案済（proposed）→ 契約中（active）→ 解約済（churned）

各段階で「反応なし」→ 再アプローチリスト（re_approach）
「興味なし」明言 → 除外（excluded）
```

## Gmail 問い合わせ処理フロー

使用メールアドレス：novatech.siteon@gmail.com

**問い合わせ経路**

| 経路 | 場面 |
|------|------|
| Gmail | サイトのフォーム（Formspree）からの通知 |
| Gmail | Coconala・Lancersからの案件通知 |
| Gmail | 直営業への返信 |
| Instagram DM | SNS経由の問い合わせ |
| LINE | 既存クライアントからの連絡 |

**処理フロー**

```
問い合わせメール着信
  ↓
Claude Code（CEO）が /gws-gmail で検知・分類
  ↓
A: 新規制作相談
   → 顧客JSONに登録 → Sheets追加
   → 営業・リサーチ部が返信文生成 → Yuichi通知
B: 既存クライアント更新依頼
   → 該当情報表示 → 対応方法提案（制作・品質部）→ Yuichi通知
C: Coconala・Lancers案件通知
   → 要約 → SITEONアップセル可否判定（営業・リサーチ部）
   → 提案文生成 → Yuichi通知
D: その他（営業メール・スパム等）
   → 重要度判定 → 不要はスキップ
  ↓
Yuichi確認→承認 → Claude Code が送信
```

**Formspreeからの通知**
NovaTech/SITEONへの問い合わせのみをGmailで管理。サロンへの予約問い合わせはクライアント自身が対応。

**生成するメール文面の種類（6種）**
問い合わせ返信 / フォローメール / 納品完了通知 / 請求案内 / 契約更新案内（12ヶ月満了時）/ 保守報告
