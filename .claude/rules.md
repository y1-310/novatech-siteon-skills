# 品質ルール（8カテゴリ・45項目）

## 1. Codex連携（5項目）

1. コード・コンテンツの記述は全てCodexに委任する
2. Codexへの指示には該当する全ルールを含める
3. Codexの出力は必ずルール準拠を確認してから採用する
4. 1セクション単位で指示を出す（一括で全体を頼まない）
5. 判断が必要な場面ではCodexに判断させず、Claude Codeが決定する

## 2. ファイル構成（4項目）

6. 単一HTMLファイル（CSS/JS inline）で構成する（スタンダード時）
7. 画像プレースホルダーは placehold.co を使用する
8. コメントは日本語で記述する
9. ファイル冒頭に `<!-- Generated by Codex for NovaTech -->` コメントを入れる

## 3. HTML品質（8項目）

10. セマンティックHTMLを使用する（header/nav/main/section/footer）
11. 全imgタグにalt属性を設定する（日本語）
12. 全imgタグにwidth・height属性を設定する
13. loading="lazy" を設定する（ヒーロー画像を除く）
14. lang="ja" をhtml要素に設定する
15. 英語要素には lang="en" を個別指定する
16. フォームのlabelとinputを紐付ける
17. noscript対応を実装する（フェードイン→常時表示 / ナビ→フル表示 / マーキー→静止 / タブ→全展開）

## 4. CSS品質（12項目）

18. CSS変数を統一使用する（--bg / --bg-alt / --bg-dark / --text / --text-mid / --text-light / --accent / --line / --r）
19. CSS変数の値をハードコードしない
20. **禁止フォント：Inter / Arial / Roboto / Helvetica は使用禁止**
21. アニメーションは transform と opacity のみ使用する（top/left/marginのアニメーション禁止）
22. セクション間padding: 120px（desktop）/ 72px（mobile）
23. border-radius は var(--r) で統一する
24. 横線で区切る — セクション間に 1px solid var(--line)
25. 彩度を抑える — ビビッドな色は使わない
26. prefers-reduced-motion に対応する
27. !important は使用禁止（noscript内を除く）
28. 日本語フォントの全ウェイト読み込み禁止（最大3ウェイト：300/400/500）
29. CSS/JSの外部ファイル分離禁止（スタンダード時）

## 5. アクセシビリティ（8項目）

30. スキップナビゲーションを設置する
31. aria-expanded・aria-label を適切に設定する
32. フォーカストラップを実装する（モーダル・メニュー）
33. focus-visible を設定する
34. コントラスト比 4.5:1 以上（WCAG 2.1 AA準拠）
35. タップターゲット 44px 以上
36. 色だけに依存しない情報伝達
37. フォームにlabel紐付け必須

## 6. パフォーマンス（6項目）

38. Lighthouse 全項目 90点以上を目標とする
39. Google Fonts に preconnect を設定する
40. font-display: swap を必ず設定する
41. 日本語フォントは最大3ウェイトまで
42. fallbackフォントスタック必須
43. ヒーロー画像以外は loading="lazy"

## 7. SEO（7項目）

44. JSON-LD 構造化データを設定する（業態別 @type：HairSalon / BarberShop / NailSalon / BeautySalon）
45. openingHoursSpecification / geo / aggregateRating / hasMap / FAQPage を含める
46. OGP 4種（og:title / og:description / og:image / og:url）+ twitter:card を設定する
47. canonical URL を設定する（マルチページ時）
48. title パターン：`[サロン名]｜[エリア名] [最寄り駅]の[業態]｜[特徴キーワード]`
49. meta description を設定する
50. viewport を設定する

## 8. NovaTech/SITEON共通（14項目）

51. 予約ボタンを5箇所に配置する（ヘッダーナビ内 / ヒーロー内 / メニューセクション下 / フッターCTA / モバイルスティッキー）
52. モバイルスティッキーCTA + padding-bottom を設定する
53. セクション区切り線：1px solid var(--line)、max-width: var(--mw)
54. セクションナンバリング表記を統一する（01, 02, 03...）
55. マーキーテキスト：テキスト2回繰り返し、CSS animation（25s linear infinite）、ディスプレイフォント italic
56. 料金は税込表記で統一する
57. tel: リンクを設定する（電話番号）
58. noscript対応を実装する
59. prefers-reduced-motion に対応する
60. 写真が主役 — テキストは添え物
61. 余白で語る — 十分な余白を確保する
62. ストーリーで巻き込む — 哲学・想いを詩的に語る
63. プライバシーポリシーページ（フォーム設置時必須）
64. 特定商取引法表記（EC時必須）

## 9. 日本語コピー品質（15項目）

全テキスト出力は `_common/japanese-copy-guide.md` に従うこと。
Codex/Kimiへの指示にもこのガイドの遵守を含めること。

### 必須チェック項目
- 文体レベル（A/B/C）が業態デフォルトに合っているか
- 1セクション内で文末表現（ですます/である）が混在していないか
- 同じ主語を2回以上繰り返していないか
- 1文に「〜的」が2回以上ないか
- 接続詞（また/さらに/そして）が連続していないか
- 「最高」「究極」等の誇張語が1ページ2回以上ないか
- 英語直訳パターン（「次のレベルへ」等）がないか
- 1文が60文字を超えていないか。見出しが20文字を超えていないか
- 「させていただく」が1ページ3回以上ないか
- 漢字ひらがな基準に違反していないか（「下さい→ください」等）
- 禁止カタカナ（ソリューション/オプティマイズ等）が使われていないか
- 単語の途中や助詞の前で改行が発生していないか
- word-break: keep-all が見出し・コピーに設定されているか
- コンセプト文が具体性テストに合格するか（サロン名を隠しても特定できるか）
- 読点が適切か（20文字以上読点なし、1文に読点3個以上がないか）

### 業態別文体デフォルト
| 業態 | 文体レベル |
|------|:---------:|
| lp-salon | A（丁寧・上品） |
| lp-nail-esthe | A |
| lp-barber | B（カジュアル） |
| lp-restaurant | B |
| lp-salon-group | A〜C |
| lp-corporate | C（信頼・格式） |
| lp-architect | C |
| NovaTech自社 | A寄りのB |

## 10. 画像品質（12項目）

全画像の選定・配置は `_common/image-guide.md` に従うこと。
Codex/Kimiへの指示にもこのガイドの遵守を含めること。

### 必須チェック項目
- 写真の4役割（空気/証拠/技術/人物）を意識し、1セクション1役割を原則としているか
- 配置パターン（P1〜P5）が適切に選択されているか
- 業態別の写真トーン・構図に従っているか
- ディテール写真が最低2枚含まれているか
- ストック写真はディテール写真のみに限定されているか
- AI生成写真がスタッフ・料理・建築に使われていないか
- 全画像にalt属性（日本語・30文字以内・具体的・体言止め）が設定されているか
- 全画像にwidth/height属性が指定されているか
- ヒーロー以外にloading="lazy"が設定されているか
- OGP画像（1200×630px）が設定されているか
- placehold.coの背景色がCSS変数（--bg）に合っているか
- 全写真の色温度が統一されているか

## 禁止事項まとめ

- Inter / Arial / Roboto / Helvetica の使用
- top / left / margin のアニメーション
- !important（noscript除く）
- 日本語フォント全ウェイト読み込み
- CSS/JSの外部ファイル分離（スタンダード時）
- CSS変数の値のハードコード
- ビビッドな色の使用
- デザインの勝手な変更
- 指示にないフォント・色・レイアウトの追加
