# 品質ルール（8カテゴリ・45項目）

## 1. Codex連携（5項目）

1. コード・コンテンツの記述は全てCodexに委任する
2. Codexへの指示には該当する全ルールを含める
3. Codexの出力は必ずルール準拠を確認してから採用する
4. 1セクション単位で指示を出す（一括で全体を頼まない）
5. 判断が必要な場面ではCodexに判断させず、Claude Codeが決定する

## 2. ファイル構成（4項目）

6. 単一HTMLファイル（CSS/JS inline）で構成する（スタンダード時）
7. 画像プレースホルダーは placehold.co を使用する
8. コメントは日本語で記述する
9. ファイル冒頭に `<!-- Generated by Codex for NovaTech -->` コメントを入れる

## 3. HTML品質（8項目）

10. セマンティックHTMLを使用する（header/nav/main/section/footer）
11. 全imgタグにalt属性を設定する（日本語）
12. 全imgタグにwidth・height属性を設定する
13. loading="lazy" を設定する（ヒーロー画像を除く）
14. lang="ja" をhtml要素に設定する
14a. `<meta name="color-scheme" content="light">` を必ず設定する（Chrome強制ダークモード対策）
15. 英語要素には lang="en" を個別指定する
16. フォームのlabelとinputを紐付ける
17. noscript対応を実装する（フェードイン→常時表示 / ナビ→フル表示 / マーキー→静止 / タブ→全展開）

## 4. CSS品質（13項目）

18. CSS変数を統一使用する（--bg / --bg-alt / --bg-dark / --text / --text-mid / --text-light / --accent / --line / --r）
19. CSS変数の値をハードコードしない
20. **禁止フォント：Inter / Arial / Roboto / Helvetica は使用禁止**
21. アニメーションは transform と opacity のみ使用する（top/left/marginのアニメーション禁止）
22. セクション間padding: 120px（desktop）/ 72px（mobile）
23. border-radius は var(--r) で統一する
24. 横線で区切る — セクション間に 1px solid var(--line)
25. 彩度を抑える — ビビッドな色は使わない
26. prefers-reduced-motion に対応する
27. !important は使用禁止（noscript内を除く）
28. 日本語フォントの全ウェイト読み込み禁止（最大3ウェイト：300/400/500）
29. CSS/JSの外部ファイル分離禁止（スタンダード時）
31. word-break: keep-all を適用する要素は、親コンテナの幅制限を確認する。狭いカード（300px以下）内では overflow-wrap: anywhere を併用する。テキストカードのグリッドは最大3列。
32. **`.section-heading` の `align-items` は必ず `start` を使用する（`end` 禁止）**。`end` を使うと段落がh2より上に浮き上がるレイアウトバグが発生する。
33. **portrait / profile 画像には必ず `max-height: 420px` を設定する**。列幅が広いと縦に伸びすぎてバランスが崩れる。
34. **2カラムグリッド（company-grid / contact-grid 等）は `align-items: stretch` を使用する**。`start` では左右の下端が揃わない。
35. **全 `<img>` タグに `width` と `height` 属性を必ず設定する**（hero含む）。未設定は CLS（累積レイアウトシフト）の原因になり Lighthouse スコアを低下させる。

## 5. アクセシビリティ（8項目）

31. スキップナビゲーションを設置する
32. aria-expanded・aria-label を適切に設定する
33. フォーカストラップを実装する（モーダル・メニュー）
34. focus-visible を設定する
35. コントラスト比 4.5:1 以上（WCAG 2.1 AA準拠）
36. タップターゲット 44px 以上
37. 色だけに依存しない情報伝達
38. フォームにlabel紐付け必須

## 5b. アニメーションルール（画像・要素別）

> 原則：**アニメーションの唯一の正当な理由は「視線誘導」**。
> 写真は静止している方が質・信頼感が高く見える。動かす理由がない要素には付けない。

### 画像へのアニメーション：原則禁止

以下の要素への `fade-in` / `translate` 付与を**禁止**する：

- `img` タグ全て
- `figure` / `.image-card` / `.gallery-item`
- `.service-card` / `.strength-card` / `.works-card` / `.gallery-card`
- `.profile-card` / `.portrait-stack` / `.member`
- `.stat-card`（カード全体）
- `.company-gallery` 内の全要素

**禁止理由**：
- `translateY` → グリッド内で隣接要素と視覚的に衝突する（今回発生した崩れの原因）
- 写真・作品は静止している方が重厚感・プロフェッショナル感が出る
- グリッド内で全カードが同時に動くと視線が散乱し、誘導効果がない

### アニメーションを付けて良い要素（限定許可）

| 要素 | 許可アニメーション | 備考 |
|------|-----------------|------|
| セクション見出し `h2` | `opacity` + `translateY(16px)` | 読む順番を作る |
| サブコピー・本文 `p` | `opacity` + `translateY(12px)` | h2より少し遅延させる |
| `section-kicker` | `opacity` のみ | |
| CTAボタン | hover時の `transform: scale` のみ | 既存ルール通り |
| ヒーロー内キャッチコピー | `opacity` + `translateY(20px)` | 最初の登場演出として有効 |
| ナビ・モーダルの開閉 | `opacity` + `transform` | UIフィードバック |

### 例外：単独画像の opacity フェード（条件付き許可）

以下を**全て**満たす場合のみ `opacity` フェード（translateY なし）を許可：
1. グリッド外の単独配置（1カラムで画面幅の50%以上を占める画像）
2. `opacity` のみ（`translateY` 禁止）
3. トランジション 0.8s 以内

### アニメーションが有効なシチュエーション（設計判断の参考）

- ✅ ページ初回表示時のヒーロー文字の登場
- ✅ スクロールで新しいセクションの見出しが現れる
- ✅ 統計数値セクションのカウントアップ
- ✅ フォームの送信完了メッセージの表示
- ✅ ハンバーガーメニューの開閉
- ❌ 既にスクロールして見える位置にある画像
- ❌ グリッド内の複数カードが一斉に動く
- ❌ 作品・料理・建築写真（静止している方が価値が伝わる）

## 9. 日本語コピー品質（15項目）

全テキスト出力は `_common/japanese-copy-guide.md` に従うこと。
Codex/Kimiへの指示にもこのガイドの遵守を含めること。

### 必須チェック項目
- 文体レベル（A/B/C）が業態デフォルトに合っているか
- 1セクション内で文末表現（ですます/である）が混在していないか
- 同じ主語を2回以上繰り返していないか
- 1文に「〜的」が2回以上ないか
- 接続詞（また/さらに/そして）が連続していないか
- 「最高」「究極」等の誇張語が1ページ2回以上ないか
- 英語直訳パターン（「次のレベルへ」等）がないか
- 1文が60文字を超えていないか。見出しが20文字を超えていないか
- 「させていただく」が1ページ3回以上ないか
- 漢字ひらがな基準に違反していないか（「下さい→ください」等）
- 禁止カタカナ（ソリューション/オプティマイズ等）が使われていないか
- 単語の途中や助詞の前で改行が発生していないか
- word-break: keep-all が見出し・コピーに設定されているか
- コンセプト文が具体性テストに合格するか（サロン名を隠しても特定できるか）
- 読点が適切か（20文字以上読点なし、1文に読点3個以上がないか）
