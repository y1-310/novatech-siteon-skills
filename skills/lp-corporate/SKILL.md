# lp-corporate — 企業LP（士業・コンサル・IT・クリエイティブ・小規模法人）

## 対象
士業事務所（弁護士・税理士・社労士等）、コンサルティング会社、小規模法人、スタートアップ、クリエイティブ系会社。従業員30名以下。

## 参考サイト
シンプル法律事務所, 大見税理士事務所, 宮沢社会保険労務士事務所, エル・コンサルティング, 合同会社キッカケ, 合同会社MMPR

## 起動条件
corporate-interview から業態判定で呼び出される。
ヒアリング結果（JSON）を受け取り、プラン判定→参照ファイル分岐→サイト生成。

```
プロ         → 上記 + multipage.md
プレミアム   → 上記 + multipage.md + premium.md
```

## 生成フロー

```
1. ヒアリングJSON受け取り
2. プラン判定（Q0.5）
3. カラープリセット決定（Q1 / Q13×Q14）
4. 各セクションのパターン決定（Q15, Q16, Q17, Q9.5, Q12）
5. オプションセクションのON/OFF（Q18）
6. 参照ファイル読み込み（プラン別）
7. Codexにセクション単位で指示→生成
8. 結合→品質チェック（rules.md準拠）
9. /seo-check + /web-design-reviewer + Lighthouse
10. プレビュー提示→修正ループ
```

## デザイン原則

1. **信頼を先に伝える** — セクション順・肩書き・実績で不安を減らす
2. **情報量で逃げない** — 必要情報は削らず、整理で見せる
3. **論理的に読ませる** — 見出し・要約・定義リストで理解負荷を下げる
4. **代表者の顔を出す** — 写真・経歴・資格で信頼を補強する
5. **CTAは相談導線** — 問い合わせ・資料請求・無料相談を明確に配置する

## 注意事項（v1.19追加）
- 士業サイトは「信頼」が最重要。過度なアニメーション禁止
- 代表者の顔写真は強く推奨。ただしオーナーの意向を最優先
- 守秘義務に配慮。お客様の声・事例は全て匿名化ガイドラインに従う

## Codexへの指示テンプレート

```
【セクション名】{セクション}
【プリセット】{カラープリセット名}
【パターン】{選択されたパターン}
【ヒアリング情報】
{該当するヒアリング回答}

以下のルールに従ってHTMLを生成してください：
- CSS変数を使用（--bg, --accent等）。ハードコードしない
- セマンティックHTML（section要素）
- 全imgにalt（日本語）、width、height、loading="lazy"（ヒーロー除く）
- コメントは日本語
- Inter/Arial/Roboto/Helvetica禁止
- アニメーションはtransform+opacityのみ
```
