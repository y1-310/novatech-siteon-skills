# lp-restaurant multipage

## 個別ページ

| ページ | URL例 | 内容 | SEO | パンくず |
|--------|-------|------|-----|---------|
| 1. Menu詳細 | `/menu/` | カテゴリ別一覧、価格、説明、写真、アレルギー注記、季節限定 | `メニュー名 + 店名 + エリア` | `Home > Menu` |
| 2. Chef紹介 | `/chef/` | シェフまたは店主の経歴、哲学、食材への考え、写真 | `シェフ紹介 + 店名` | `Home > Chef` |
| 3. Event詳細 | `/event/{slug}/` | イベント概要、開催日、定員、料金、予約方法 | `イベント名 + 店名 + 開催日` | `Home > Event > Detail` |
| 4. Blog記事 | `/blog/{slug}/` | コラム、季節メニュー告知、営業案内、裏話 | `記事タイトル + 店名` | `Home > Blog > Article` |
| 5. Course詳細 | `/course/` | コース名、金額、料理数、飲み放題有無、予約条件 | `コース料理 + 店名` | `Home > Course` |

## ルーティング方針

| 項目 | 内容 |
|------|------|
| 下層ページ | `/slug/` の静的ルーティングを基本とする。 |
| イベントとブログ | 一覧ページ省略も許容するが、導線上の一貫性を保つ。 |
| トップから下層への導線 | Hero、Menu、Footer のいずれかに置く。 |

### ルーティング例

```text
/
/menu/
/chef/
/event/{slug}/
/blog/{slug}/
/course/
```

## SEO方針

| 項目 | 内容 |
|------|------|
| 各ページ | 固有の title / description / OGP を持つ。 |
| canonical | 個別URLごとに設定する。 |
| メニュー、イベント、記事 | 重複文言を避け、ページ固有情報を優先する。 |

## パンくず

| 項目 | 内容 |
|------|------|
| 下層ページ | すべての下層ページにパンくずを配置する。 |
| 構造化データ | パンくずは構造化データ `BreadcrumbList` と整合させる。 |

## JSON-LD

| ページ | 型・方針 |
|--------|---------|
| Menu詳細 | `Menu` または `hasMenu` を検討する。 |
| Blog記事 | `BlogPosting` |
| Event詳細 | `Event` |
| Chef紹介 | `Person` |
| Course詳細 | `Menu` または `Offer` を補助的に使う。 |
