# lp-architect — 建築事務所・工務店・リノベーション会社

## 対象
建築事務所・工務店・リノベーション会社。個人〜15名程度の小規模事業者。

## 参考サイト
Kocochi Architect, NISHIKEN architect design, marutaHOUSE, Toivo, STUDIO YY

## 起動条件
architect-interview から業態判定「設計事務所 / 工務店 / リノベーション会社 / その他」で呼び出される。
ヒアリング結果（JSON）を受け取り、プラン判定→参照ファイル分岐→サイト生成。

## プラン判定→参照ファイル分岐

```
スタンダード → SKILL.md + design-system.md + sections.md
プロ         → 上記 + multipage.md
プレミアム   → 上記 + multipage.md + premium.md
```

## 共通参照ファイル

```

```

## 生成フロー

```
1. ヒアリングJSON受け取り
2. プラン判定（Q0.5）
3. カラープリセット決定（Q13×Q14）
4. 各セクションのパターン決定（Q15, Q16, Q17, Q17.5, Q9.5, Q12）
5. オプションセクションのON/OFF（Q18）
6. 参照ファイル読み込み（プラン別）
7. Codexにセクション単位で指示→生成
8. 結合→品質チェック（rules.md準拠）
9. /seo-check + /web-design-reviewer + Lighthouse
10. プレビュー提示→修正ループ
```

## デザイン原則

1. **Worksが主役** — 写真は実績の証明として最優先で見せる
2. **情報は簡潔に出す** — 信頼構築に必要な情報は省略しない
3. **理知的に語る** — テキストは詩的すぎず、設計意図を端的に示す
4. **素材感で魅せる** — 白・木・陰影のコントラストを丁寧に扱う
5. **線と余白で整える** — セクション間に1px solid var(--line)

### 共通演出要素

| 要素 | 仕様 |
|------|------|
| マーキーテキスト | Philosophyセクション前後に配置。英字キーワード無限スクロール。CSS animationのみ |
| スクロールフェードイン | Intersection Observer。opacity 0→1 + translateY 12px→0。duration 0.7s |
| ヘッダースクロール変化 | transparent → 半透明背景 + border-bottom + backdrop-filter blur |
| モバイルスティッキーCTA | 画面下部に資料請求 / 相談予約ボタン常時表示。padding-bottom追加 |

## Codexへの指示テンプレート

```
【セクション名】{セクション}
【プリセット】{カラープリセット名}
【パターン】{選択されたパターン}
【ヒアリング情報】
{該当するヒアリング回答}

以下のルールに従ってHTMLを生成してください：
- CSS変数を使用（--bg, --accent等）。ハードコードしない
- セマンティックHTML（section要素）
- 全imgにalt（日本語）、width、height、loading="lazy"（ヒーロー除く）
- コメントは日本語
- Inter/Arial/Roboto/Helvetica禁止
- アニメーションはtransform+opacityのみ
```
