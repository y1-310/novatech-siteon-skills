## 1. 文体レベルの統一

サイト全体で敬語の度合いを統一する。AIは特に文体混在が起きやすい。

### 3レベル定義

#### レベルA：丁寧・上品

```
用途：女性向けサービス。安心感と品格を重視。
見出し：体言止め or 名詞句（「髪と心に、やすらぎを。」）
本文末：〜しています / 〜いただけます / 〜ございます
CTA：「ご予約はこちら」「お気軽にどうぞ」
注意：「いたします」は多用しない。「〜しています」で十分。
```

#### レベルB：カジュアル・親しみ

```
用途：男性向け、飲食店。親しみやすさと端的さを重視。
見出し：短文・断言（「男の身だしなみ、ここで整う。」）
本文末：〜します / 〜できます / 〜です
CTA：「予約する」「LINEで相談」
注意：くだけすぎない。「〜じゃん」「〜だよね」は禁止。
```

#### レベルC：信頼・格式

```
用途：士業、建築、法人。専門性と信頼を重視。
見出し：格調ある表現（「確かな技術で、暮らしを設計する。」）
本文末：〜いたします / 〜しております / 〜と考えています
CTA：「お問い合わせ」「ご相談ください」
注意：硬すぎない。「弊社」より「私たち」の方が温かい。
```

### 業態別デフォルトレベル

| 業態 | デフォルト | 理由 |
|------|:---------:|------|
| lp-salon | A | 女性客。安心感と上品さ |
| lp-nail-esthe | A | 同上 |
| lp-barber | B | 男性客。端的で力強い言葉 |
| lp-restaurant | B | 親しみやすさ重視 |
| lp-salon-group | A〜C | 企業規模による。個人寄りならA、法人寄りならC |
| lp-corporate | C | 士業・法人。信頼感重視 |
| lp-architect | C | 専門性と格調 |
| NovaTech自社サイト | A寄りのB | サロンオーナーに寄り添う温かさ＋サービスの信頼感 |

## 2. 文末混在禁止

1セクション内で敬語レベルが異なる文末を混ぜない。

```
× 「施術を行っております。気軽にご連絡ください。」
  → 「おります」（謙譲語）と「ください」（尊敬語）のレベル差

○ 「施術を行っています。お気軽にご連絡ください。」
  → 「ています」＋「お〜ください」で統一

× 「最高の仕上がりを提供する。お客様の笑顔が私たちの喜びです。」
  → 「する」（である調）と「です」（ですます調）が混在

○ 「最高の仕上がりを提供します。お客様の笑顔が私たちの喜びです。」
```

### 見出しと本文の文体は変えてよい

```
○ 見出し：「髪と心に、やすらぎを。」（体言止め）
  本文：「当サロンでは、お客様一人ひとりに合わせた施術を行っています。」（ですます）

→ 見出しが体言止め・名詞句で、本文がですます調。これは自然。
  ただし見出しの中で「です」と体言止めを混ぜるのはNG。
```

## 3. 主語繰り返し禁止

同一セクション内で同じ主語を2回以上書かない。日本語は主語を省略する言語。

```
× 「私たちは、お客様の髪を大切にします。
   私たちは、一人ひとりに寄り添います。」

○ 「お客様の髪を大切にし、一人ひとりに寄り添います。」
```

```
× 「当サロンでは髪質改善を行っています。
   当サロンではヘッドスパもご用意しています。」

○ 「髪質改善やヘッドスパもご用意しています。」
```

---

## 4. 「〜的」「〜性」制限

1文に「〜的」「〜性」は1回まで。

```
× 「革新的な技術で、専門的な施術を、効率的に提供します。」

○ 「独自の技術で、専門の施術を、手際よく提供します。」
```

---

## 5. 接続詞連発禁止

連続する文で接続詞（また・さらに・そして・しかし・加えて）を2回以上使わない。

```
× 「また、髪質改善も行っています。さらに、ヘッドスパもご用意しています。
   そして、完全個室で施術を行います。」

○ 「髪質改善やヘッドスパもご用意。完全個室でゆったりお過ごしいただけます。」
```

## 6. 誇張語制限

「最高」「究極」「完璧」「至高」「唯一無二」は原則禁止。使うなら1ページに1回まで。
具体的な言葉に置き換える。

### 業態別の許容度

```
lp-salon / lp-nail-esthe：
  × 「最高」「究極」 → 上品さに合わない
  ○ 「特別な」「こだわりの」 → 柔らかい表現

lp-barber：
  △ 「最高の仕上がり」 → 男性向けなら許容できる場面がある
  ○ 「一切妥協しない」 → バーバーのトーンに合う

lp-restaurant：
  △ 「至福の一皿」 → 飲食では一般的。ただし多用NG
  ○ 「素材の味を、そのまま。」 → 具体的で強い

lp-architect：
  × 「究極の住空間」 → 安っぽく聞こえる
  ○ 「暮らしに寄り添う設計を。」 → 格調がある

lp-corporate：
  × 「最高のサービス」 → どの会社でも言える
  ○ 「創業から20年、○○の課題に向き合い続けています。」 → 具体的
```

### 置き換え表

| 誇張語 | 置き換え候補 |
|--------|------------|
| 最高の | こだわりの / 納得の / 自信を持って |
| 究極の | とことん追求した / 徹底した |
| 完璧な | 丁寧な / 細部まで考えた |
| 唯一無二の | ここだけの / あなただけの |
| 至高の | 格別な / 心を込めた |
| 圧倒的な | 確かな / 選ばれ続ける |

## 7. 英語直訳禁止パターン

AIが生成する日本語で頻出する英語直訳構文を禁止する。

## 11. カタカナ使用基準

### 判断の3ルール

**ルール1：「お客様が普段使っている言葉か？」テスト**

サロンに来るお客様が友達との会話で使う言葉ならOK。

```
○ 「今日カラーしてきた」 → OK
× 「ソリューション提供してもらった」 → NG
```

**ルール2：「日本語に置き換えて品質が上がるか？」テスト**

```
トリートメント → 「髪の手入れ」？ → 不自然。そのまま
リラックス → 「くつろぎ」？ → 文脈による。両方OK
ソリューション → 「解決策」？ → 自然。日本語の方がいい
```

**ルール3：「NovaTech自社サイト」と「クライアントサイト」で基準を分ける**

NovaTechのターゲットはサロンオーナー。サロンサイトのターゲットはお客様。

## 12. 改行：文節ルール

日本語には文節（意味のかたまり）があり、文節の途中で改行すると読みにくい。

### 禁止

```
1. 単語の途中で改行しない
   × マンツー｜マン
   × トリート｜メント
   × ホーム｜ページ

2. 助詞の前で改行しない
   × 心に寄り添う｜の施術
   × お客様｜が安心できる

3. 助詞だけを次の行に送らない
   × 美しさ
     を引き出す
```

### 推奨

```
1. 読点（、）の直後で改行する
   ○ お客様の髪と心に寄り添う、
     マンツーマンの施術。

2. 句点（。）の直後で改行する
   ○ まずはカウンセリングから。
     髪質に合わせたメニューをご提案します。

3. 文節の区切りで改行する
   ○ あなたのサロンには、
     あなただけの物語がある。
```

### 改行注意ワード（長いカタカナ複合語）

```
トリートメント（7文字）
バレイヤージュ（7文字）
インナーカラー（7文字）
プライベートサロン（10文字）
マンツーマン（6文字）
ヘッドスパ（5文字）
→ これらは途中で切らない。CSSで制御する。
```

## 13. 改行：CSS実装

```css
/* 見出し・コピー：文節途中の改行を防止 */
h1, h2, h3,
.hero-copy,
.concept-text,
.section-title {
  word-break: keep-all;
  overflow-wrap: break-word;
}

/* 見出しの行長制限（長すぎる行を防ぐ） */
h1, .hero-copy {
  max-width: 18em;
}

h2 {
  max-width: 22em;
}

/* 本文：通常の日本語改行 */
p, li, dd {
  word-break: normal;
  overflow-wrap: break-word;
  line-height: 2;
}

/* 強制改行が必要な箇所 */
/* HTMLで <br> を明示指定する。CSSの改行に任せない。 */
```

### Codexへの指示テンプレートに含める文

```
日本語の文節を意識し、単語の途中や助詞の前で改行が発生しないようにすること。
CSSに word-break: keep-all を設定し、見出しには max-width を指定すること。
見出し内で意図的に改行する場合は <br> で明示的に指定すること。
```

### カード・狭コンテナ内の改行制御

`word-break: keep-all` は行の途中での改行を防ぐが、カード等の幅が狭いコンテナ（300px以下）では
テキストがコンテナからはみ出す原因になる。以下のルールで対処する。

#### ルール

1. **カード内の見出し（h3等）**: `word-break: keep-all` + `overflow-wrap: anywhere`
   - `anywhere` は `break-word` より強力で、はみ出しを確実に防ぐ
   - 文節の途中で改行される可能性はあるが、はみ出すよりは遥かにマシ
2. **カード内の本文（p）**: `word-break: normal`（デフォルト）を維持
3. **グリッド列数の目安**: テキストカードは最大3列。5列以上は見出しが収まらない
4. **テスト必須**: `word-break: keep-all` を適用した要素は、モバイル幅（375px）でテキストがはみ出さないことを必ず確認する

#### CSS実装

```css
/* カード・狭コンテナ内の見出し */
.card h3,
.problem-card h3,
[class*="-card"] h3 {
  word-break: keep-all;
  overflow-wrap: anywhere;
}

/* グリッドカードのテキスト溢れ防止 */
.card,
[class*="-card"] {
  overflow: hidden; /* 万が一のはみ出し防止 */
}
```

#### Codex / Kimi への注意

```
word-break: keep-all を使う場合、親コンテナの幅を確認すること。
幅300px以下のカードでは overflow-wrap: anywhere を併用すること。
テキストカードのグリッドは最大3列にすること。5列はNG。
```

## 14. 具体性テスト

AIが生成するコピーで最も多い問題は「どのサロンにも当てはまる汎用コピー」。

### テスト方法

**コピーを読んで、サロン名を隠しても「あのサロンだ」と特定できるか？**

```
× どのサロンにも使える：
  「上質な空間で、特別なひとときを。」
  「あなただけの美しさを引き出します。」
  「心と髪を、美しく。」

○ そのサロンにしか言えない：
  「乾かしただけでまとまる。」（Kicchi）
  「白髪をコンプレックスから『魅力』に変える。」（Amie）
  「心が満たされていく、そんな時間をもてるサロン。」（tukinone）
```

### 汎用コピー禁止ワードリスト

以下の語句だけで構成されたコピーは却下する：

```
上質 / 特別 / こだわり / 寄り添う / 輝く / 最高 / 至福 /
あなただけの / 本物の / 贅沢な / 極上の / 洗練された
```

**これらの語句自体は禁止ではない。** ただし、これらだけで作られたコピーは
具体性がゼロなので禁止。具体的な情報と組み合わせて使う。

```
× 「上質なこだわりで、特別な時間を。」 → 具体性ゼロ
○ 「京町家の個室で、アロマとともに過ごす貸切の時間。」 → 具体的
```

### Codex / Kimi への指示テンプレート

```
コンセプト文を書く際は、ヒアリングで得た「このサロンにしかない特徴」を
必ず含めること。
「上質」「特別」「こだわり」だけで構成された汎用コピーは禁止。
オーナーが実際に使った言葉を優先的に採用すること。
具体性テスト：サロン名を隠しても特定できるコピーを目指すこと。
```

## 15. 読点の打ち方

AIは読点が少なすぎるか多すぎる。基準を明確にする。

### ルール

```
1. 20文字以上続いたら読点を打つ
2. 「〜て」「〜し」「〜が」の後に読点を入れる
3. 1文に読点は最大2つ。3つ以上入るなら文を分割する
```

### 例

```
× 読点なし：
  「お客様の髪質やライフスタイルに合わせた施術を丁寧にカウンセリングを
   行いながら最適なメニューをご提案します。」
  → 息ができない

× 読点多すぎ：
  「お客様の、髪質や、ライフスタイルに、合わせた、施術を、ご提案します。」
  → 途切れすぎ

○ 適切：
  「お客様の髪質やライフスタイルに合わせて、
   最適なメニューをご提案します。」
  → 1文に読点1つ。自然に読める
```

## チェックリスト（最終確認用）

サイト公開前に全テキストを以下でチェックする：

- [ ] 文体レベル（A/B/C）が全セクションで統一されているか
- [ ] ですます調とである調が混在していないか
- [ ] 同じ主語が2回以上繰り返されていないか
- [ ] 1文に「〜的」が2回以上ないか
- [ ] 接続詞が連続していないか
- [ ] 「最高」「究極」等が1ページ2回以上ないか
- [ ] 英語直訳パターンがないか
- [ ] 60文字を超える文がないか
- [ ] 見出しが20文字を超えていないか
- [ ] 「させていただく」が3回以上ないか
- [ ] 漢字ひらがな基準に違反していないか（「下さい」→「ください」等）
- [ ] 禁止カタカナが使われていないか
- [ ] 単語の途中で改行が発生していないか（モバイル実機確認）
- [ ] コンセプト文が具体性テストに合格するか
- [ ] 読点が適切に打たれているか（20文字以上読点なし、1文に3個以上 がないか）
