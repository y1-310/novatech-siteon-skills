## 1. 写真の4つの役割

優れたサイトでは、全ての写真が以下の4つの役割のいずれかに分類される。
1セクション内では1つの役割に統一するのが原則。混ぜる場合はメイン（大）とサブ（小）で役割を分ける。

| 役割 | 目的 | 例 |
|------|------|-----|
| A. 空気写真 | 場の雰囲気・世界観を伝える | 内装の引き写真、自然光の空間 |
| B. 証拠写真 | 「本当にこういう場所です」を証明する | 店舗写真、施術結果、完成建築 |
| C. 技術写真 | 腕前・専門性を見せる | 道具の俯瞰、施術工程、設計図 |
| D. 人物写真 | 「この人に任せたい」を作る | スタッフ紹介、オーナーストーリー |

## 2. 写真の5つの配置パターン

| パターン | 構造 | 適するセクション |
|---------|------|----------------|
| P1：2枚並列 | 写真2枚を横に並べ、下にテキスト | Concept, Salon, Menu |
| P2：1枚大＋テキスト横 | 写真1枚（大）の横にテキスト。2カラム | Features, About |
| P3：写真1枚全幅 | 写真が幅いっぱい。テキストは上 or 下 | Hero, CTA背景 |
| P4：写真グリッド | 4〜6枚をグリッドで並べる | Gallery, Works, 店舗紹介 |
| P5：写真＋引用 | 写真の横に口コミ or コンセプト文 | Testimonials, Owner |

Codexへの指示では「ConceptセクションはP1で」のようにパターン番号で指定する。

## 3. 写真密度の基準

| 目的 | 密度 | 1セクション枚数 | 例 |
|------|------|:-:|------|
| 高級感 | 低い | 1〜2枚 | tukinone, をとわ |
| 信頼感 | 中程度 | 2〜4枚 | Kicchi, 日本の住まい |
| 親しみ | 高い | 3〜6枚 | DILLY DALLY, Amie |

プラン別枚数：スタンダード6〜10枚、プロ15〜25枚、プレミアム25〜40枚。

## 4. ディテール写真のルール

全サイトでディテール写真を最低2枚確保する。ストック写真を使ってよいのはディテール写真のみ。

| 業態 | ディテール写真の例 |
|------|------------------|
| salon | アロマオイル、ドライフラワー、タオル、木の床 |
| barber | ストレートレーザー、シェービングブラシ、革のチェア |
| nail-esthe | ジェルのボトル、リング、花びら、施術道具 |
| restaurant | 器、箸置き、薪、パン断面、ハーブ |
| architect | 木材テクスチャ、コンクリート目地、植栽ディテール |
| corporate | 名刺、書類、オフィスディテール |

## 5. 画像ソース優先順位

| 優先度 | ソース | 用途 |
|:---:|--------|------|
| 1 | クライアント撮影写真 | 全用途 |
| 2 | Unsplash / Pexels | ディテール・空気写真・建築・料理 |
| 3 | 日本向けストック（PAKUTASO, photoAC, model.foto, O-DAN） | 日本人モデル・日本の空間 |
| 4 | Gemini / DALL-E 3生成 | 日本人スタッフ写真・特定業態の接客シーン（Section 8.7参照） |
| 5 | placehold.co | 開発中のみ（デモ提示以降は禁止） |

**AI生成を使ってよい場面**: 日本人・東アジア系が必要でUnsplash/日本向けストックで見つからない場合のみ（Section 8.7参照）。
**AI生成禁止**: 料理写真・建築外観写真・商品クローズアップ（質感の再現が困難なため）。

## 6. 業態別：写真トーン・構図・比率

### 全業態共通の原則

1. 「誰のため/何を売る」を最初に明示する
2. 写真の役割を4分類（空気/証拠/技術/人物）で分ける
3. 導線は業態ごとに最適化する
4. 静的でも強い。動き過ぎはNG

### 業態別の写真役割比率

| 業態 | 空気 | 証拠 | 技術 | 人物 |
|------|:---:|:---:|:---:|:---:|
| lp-salon | 35% | 30% | 10% | 25% |
| lp-barber | 25% | 40% | 20% | 15% |
| lp-nail-esthe | 30% | 45% | 5% | 20% |
| lp-restaurant | 25% | 35% | 15% | 25% |
| lp-architect | 30% | 50% | 15% | 5% |
| lp-corporate | 20% | 20% | 10% | 50% |
| NovaTech自社 | 15% | 50% | 20% | 15% |

### 6-1. lp-salon

分析元：tukinone, milkarucge, Kicchi, Amie, らふる

色味：暖色・低彩度・くすみ。光：自然光（午前〜14時）。被写体：①内装 ②スタイル ③スタッフ ④ディテール。構図：余白多め・三分割法。アスペクト比：3:4縦長。NG：蛍光灯、カメラ目線、ビビッド色。

### 6-2. lp-barber

分析元：8 BARBER SHOP, WAKAMIYA, HIRAKAWA, NAKAYA, OLDO BARBER

色味：モノクロ・セピア・低彩度（5/5件）。光：ハイコントラスト。被写体：①道具俯瞰 ②後頭部 ③内装（レンガ・革） ④Before/After。構図：直線的・対称的。アスペクト比：1:1正方形。NG：パステル、花、柔らかい光。

### 6-3. lp-nail-esthe

分析元：FASTNAIL, esNAIL, NANA, Salon de Noble

色味：明るめ・高彩度・パステル（4/4件）。光：自然光・柔らかい。被写体：①ネイルクローズアップ ②手元 ③施術空間 ④スタッフ。構図：曲線的・有機的。アスペクト比：1:1正方形。特有：白背景 or 肌色ベース。NG：暗い背景、モノクロ。サブ業態別：ネイル=デザイン4列グリッド、エステ=ビフォーアフター、まつエク=本数別比較。

### 6-4. lp-restaurant

分析元：をとわ, sasa, Humming Fields, DILLY DALLY, 北のはしベーカリー

被写体：料理60%、空間25%、人物15%。構図：料理は斜め45度。シズル感（湯気・ツヤ）を意識。NG：メニュー表的な写真。

飲食タイプ別色味：和食=暖色・暗め、イタリアン=暖色・自然、カフェ=明るめ・ナチュラル、ダイナー=ポップ、ベーカリー=ベージュ。

4類型：料理を売る型 / 空間を売る型 / 時間を売る型 / 商品思想を売る型。

### 6-5. lp-architect

分析元：色景, MON architects, S PLUS ONE, 日本の住まい, 新井アトリエ

色味：白黒〜ベージュ・低彩度。光：自然光が絶対（5/5件）。被写体：①外観 ②内観 ③ディテール ④模型。構図：正面＋パース・人物なし。特有：Works一覧は統一トーン絶対。NG：人物入り、HDR、広角歪み。

4類型：作品主導型 / 思想主導型 / 工程説明型 / 暮らし提案型。

**Worksセクション専用ルール（厳守）**:
- 実績写真は必ず**異なる物件・案件**を使用（同一建物の別角度のみは禁止）
- 各物件で「外観」「内観」「ディテール」のいずれか1役割を割り当て
- 物件ごとにトーン統一（全物件が同じ明度・彩度帯に収まること）
- 住宅・店舗・集合住宅・リノベーション等、**種別のバリエーション**を持たせる

### 6-6. lp-corporate

分析元：税理士法人SKC, 行政書士法人まるこ, 久郷行政書士, 栗原法務事務所, キロロ労務人事

色味：白＋紺/グリーン/ゴールド。被写体：①代表者の顔写真（最重要） ②事務所内装 ③資格・メディア。特有：顔出しが信頼の基本。笑顔＞真顔。NG：派手な色、写真多用。

士業別カラー：税理士=紺+金、行政書士=グリーン、社労士=オレンジ、弁護士=黒+金、コンサル=ブルー。

**代表者写真ルール（強化版）**:
- 必須条件：①正面向き ②笑顔（口角が上がっている）③スーツまたは清潔感ある服装 ④白・オフィス・ニュートラル背景
- 日本人・東アジア系の外見（Unsplashで見つからない場合はGemini生成）
- NG：横向き・暗い背景・真顔・カジュアル服装・屋外ランダム背景

## 8.5 デモサイト生成時の画像選定フロー

デモサイト（Phase 1: デモ提示）を生成する際、以下のフローで画像を選定・埋め込む。

### フロー

1. `sections.md` の `{画像}` プレースホルダーを確認
2. そのセクションの写真の「役割」（空気/証拠/技術/人物）を特定
3. 業態別の写真トーン・構図（セクション6）を確認
4. 下記の検索キーワードでUnsplash/Pexelsを検索
5. 色温度・トーンがdesign-system.mdのCSS変数と合う写真を選定
6. URL形式: `https://images.unsplash.com/photo-{id}?w={width}&h={height}&fit=crop&auto=format`
7. alt属性を日本語で記述（30文字以内・具体的・体言止め）

### 業態別 Unsplash 検索キーワード

| 業態 | 検索キーワード例 |
|------|---------------|
| lp-salon | `hair salon interior`, `hairstyle woman`, `salon tools`, `dried flowers`, `natural light salon` |
| lp-barber | `barber shop`, `barbershop tools`, `mens haircut`, `straight razor`, `leather barber chair` |
| lp-nail-esthe | `nail art`, `nail salon`, `gel nails`, `spa treatment`, `beauty salon pastel` |
| lp-restaurant | `restaurant dish`, `bistro interior`, `cooking chef`, `wine glass`, `bread bakery` |
| lp-architect | `modern house exterior`, `interior design minimal`, `architecture detail`, `natural light room`, `concrete texture` |
| lp-corporate | `business portrait smile`, `office professional`, `tax accountant`, `legal office`, `business meeting` |

### 色温度の合わせ方

- design-system.mdの `--bg` の色温度を基準にする
- 暖色系（#F9F5F0等）→ 暖色の写真を選ぶ
- 寒色系（#F5F7FA等）→ ニュートラル〜寒色の写真を選ぶ
- 同一サイト内で暖色と寒色の写真を混ぜない

### 重複チェック（必須）

- **同一photo-IDは1サイト内で1回のみ**（同じUnsplash URLを2箇所以上に使用禁止）
- **隣接セクションで同系構図の写真禁止**（例：同じ角度の室内2枚連続はNG）
- **ギャラリーセクションは全枚数ユニーク**（1枚も重複禁止）
- 生成後に `grep -c '{photo-id}' index.html` で各URLの出現回数を確認する

### 業態別最低画像数

| 業態 | 最低画像数 |
|------|-----------|
| lp-salon | 18枚以上 |
| lp-barber | 15枚以上 |
| lp-nail-esthe | 15枚以上 |
| lp-restaurant | 15枚以上 |
| lp-architect | 18枚以上 |
| lp-corporate | 12枚以上 |

### 注意

- スタッフ写真・代表者写真：Unsplashで日本人・東アジア系が見つからない場合はGemini生成を使用（Section 8.7参照）
- 料理写真・建築写真にはUnsplashの実写真を使用（AI生成禁止）
- デモサイトのfooterに「※ 写真はイメージです」を追記する
- Gemini生成画像を使用した場合は「※ 一部AI生成画像を使用しています」も追記する

## 8.6 詳細画像品質ルール

### A. 技術仕様

| 用途 | 最小サイズ | 推奨アスペクト比 | format |
|------|-----------|----------------|--------|
| Hero（全幅） | 1920×1080px | 16:9 or 3:1 | WebP/JPG |
| コンセプト・特徴 | 800×600px | 4:3 or 3:2 | WebP/JPG |
| スタッフ・人物（縦） | 400×500px | 4:5 | WebP/JPG |
| ギャラリー（縦） | 600×800px | 3:4 | WebP/JPG |
| 建築・Works | 800×600px | 16:9 or 4:3 | WebP/JPG |
| OGP | 1200×630px | 1.91:1 | JPG |

### B. 色温度・トーン整合

| カラープリセット | 選ぶべきトーン | 避けるトーン |
|----------------|--------------|------------|
| グレージュ×モード（--bg: #F5F2EE等） | 暖色・低彩度・くすみ・自然光 | 蛍光色・高彩度・青白い光 |
| ホワイト×ブルー（--bg: #FAFAFA等） | クール・明るい・清潔感 | 暖色・黄ばみ・暗い |
| ブラック×ゴールド（--bg: #1A1816等） | コントラスト強・深み | パステル・薄い色 |
| ナチュラル×グリーン | 自然光・緑・アースカラー | 人工的な光・無機質 |

同一サイト内で暖色と寒色の写真を混在させない。

### C. 人物写真ルール

- **日本人・東アジア系の外見を必ず使用**
  - Unsplashキーワード: `japanese woman`, `asian stylist`, `asian professional`, `japanese businesswoman`
  - 見つからない場合はGemini生成を使用（Section 8.7参照）
- **過度に演出された"広告モデル"禁止**（自然な表情・シーンを選ぶ）
- **スタッフ写真は全員異なる人物**（同一人物の別カット使い回し禁止）
- **背景がブランドトーンと合うものを選ぶ**

### D. セクション内容との一致ルール（最重要）

各写真のalt属性がそのセクションのコピーと論理的に一致していること。

| セクションコピーの主張 | 適切な写真 | NG写真 |
|---------------------|-----------|--------|
| 「カウンセリング重視」 | 美容師が客に向き合っている写真 | ハサミのみのクローズアップ |
| 「素材へのこだわり」 | 食材・素材のクローズアップ | 店内の引き写真 |
| 「施工実績」 | 複数の異なる建物・物件 | 同一建物の別角度のみ |
| 「信頼・実績」 | スーツ着用・書類・ミーティングシーン | カフェや無関係な風景 |
| 「髪質改善」 | 施術後の艶やかな髪のクローズアップ | 店内インテリアのみ |

### E. 使用前チェックリスト（必須）

```
□ このphoto-IDは同一サイト内で初めて使用するか？
□ セクションのコピー内容と写真の被写体が一致しているか？
□ ブランドの色温度・トーンと合っているか？
□ 人物が写っている場合、日本人・東アジア系か？
□ altテキストが写真の内容を具体的に表しているか？（体言止め30字以内）
□ 隣接セクションと構図・色が被っていないか？
□ 業態別最低画像数を満たしているか？
```

## 8.7 Gemini生成画像使用ガイドライン

### 使用する場合（Gemini推奨シーン）

| シーン | 理由 |
|--------|------|
| スタッフ・人物写真（日本人指定） | Unsplashで東アジア系が極端に少ない |
| 日本人×業態の接客シーン | 「日本人美容師とお客様」等はUnsplashでほぼ存在しない |
| ブランドカラーに完全一致させたい背景・小物 | Unsplashでは色温度が合うものが限定的 |
| 架空店舗の「内装イメージ」を特定雰囲気で作りたい | 実在店舗写真では過度に具体的になりすぎる |

### 使用しない場合（Unsplash優先）

| シーン | 理由 |
|--------|------|
| 料理・食材のクローズアップ | 実写の方が質感・食欲感が上。AI生成は不自然になりやすい |
| 建築・外観・施工写真 | Unsplashの建築写真は品質が高く、AIより自然 |
| 自然・風景・空 | Unsplashが圧倒的に豊富で高品質 |
| Hero画像（全幅） | Unsplashの方が解像度・品質が安定している |
| 商品・素材のクローズアップ | 質感の再現はUnsplash実写が優れる |

### 使用判断フロー

```
[必要な写真を定義]
        ↓
人物が写っているか？
  Yes → 日本人・東アジア系が必要か？
          Yes → Unsplashで "japanese" / "asian" キーワード検索
                  3件以内で見つかる？
                    No  → Gemini生成（下記プロンプトルール参照）
                    Yes → Unsplash使用
          No  → Unsplash使用
  No  → 料理・建築・自然・風景か？
          Yes → Unsplash優先（Gemini禁止）
          No  → Unsplashで3件以内に見つかる？
                  No  → Gemini生成
                  Yes → Unsplash使用
```

### Geminiプロンプトルール

**必須要素**:
- 「フォトリアリスティック」「リアルな写真風」を明記
- 人物の場合: 「日本人女性/男性」「自然な表情」「（業態）の（シーン）」
- 照明: 「自然光」「柔らかい光」
- 解像度: 「高解像度」「1200×800px以上相当」

**禁止要素**:
- 「イラスト」「アニメ」「水彩」「絵画風」の指定
- 過度に演出された構図（空中に浮くオブジェクト等）
- 文字・ロゴを含む指定

**プロンプト例（美容室スタッフ）**:
```
日本人女性の美容師がお客様の髪をカットしているリアルな写真。
自然光の入る白い美容室。プロフェッショナルで温かみのある雰囲気。
フォトリアリスティック、高解像度、自然な表情。
```

**プロンプト例（税理士・男性代表）**:
```
日本人男性の税理士が書類を前にほほ笑んでいるリアルな写真。
清潔感のある白いオフィス。スーツ着用。信頼感と親しみやすさ。
フォトリアリスティック、高解像度、正面向き。
```

### Gemini生成後の品質チェック（必須）

```
□ 手・指の形が自然か（AI生成で歪みやすい）
□ 背景に不自然な歪み・繰り返しパターンがないか
□ 文字・記号が混入していないか
□ 人物の顔が自然か（目・鼻・口の位置・左右対称）
□ ブランドカラーのトーンと合っているか
□ footerに「※ 一部AI生成画像を使用しています」を追記したか
```

## 8.8 ムードボード設計ルール

サイト全体を「1つの視覚世界」として設計する。個別セクションではなく**サイト全体を1枚のキャンバス**として見ること。

### A. 基準写真の定義（サイト生成前に必ず実施）

Hero画像をムードボード基準写真として定め、以下の**視覚プロファイル**を記録する：

```yaml
基準写真: {Hero画像URL}
色温度: warm / cool / neutral  （いずれか1つ）
明度: bright / mid / mid-dark / dark  （いずれか1つ）
彩度: vivid / natural / muted / desaturated  （いずれか1つ）
主な色域: #xxxxxx〜#xxxxxx  （暗→明の2色）
雰囲気3語: {形容詞} / {形容詞} / {形容詞}
撮影スタイル: lifestyle / documentary / editorial / architectural
```

### C. 全写真の選定前チェック

```
各写真を使う前に基準写真と比較:
□ 色温度が同じ帯域か？（warm←→cool の混在禁止）
□ 明度差が2段階以内か？（dark サイトに bright 写真禁止）
□ 彩度が極端に違わないか？（muted サイトに vivid 禁止）
□ 雰囲気3語と合致するか？
□ 撮影スタイルが一致するか？
```

### D. 違和感テスト（生成後）

全写真を縦に並べてスクロールする。以下が1つでも当てはまる写真は差し替え：
- 「この写真だけ別のサイトから来た」と感じる
- Heroと同じサイトとは思えない色・明るさ
- 人物の背景だけ全く別の空間
- 撮影スタイルが明らかに異なる（reportage の中に studio 1枚など）

## 8.9 ファーストビュー法則・セクション遷移ルール

### ファーストビュー法則

Hero画像だけで業態・ターゲット・雰囲気が**0.5秒で伝わること**を必須とする。

| 業態 | Heroで伝えるべきこと | NG例 |
|------|---------------------|------|
| lp-salon | 「自然光の美容室」または「施術後の美しい髪」 | 南国リゾート・都市夜景・抽象的な風景 |
| lp-barber | 「理容室の道具・椅子・男性客」 | 自然風景・花・明るいパステル |
| lp-restaurant | 「料理またはレストラン内装」 | オフィス・自然・人物のみ |
| lp-architect | 「建物の外観または内観」 | 夜の都市スカイライン（建築物でない） |
| lp-corporate | 「代表者の顔またはオフィス」 | 抽象的グラフィック・自然風景 |

### セクション遷移ルール

スクロール体験を壊さないための明度遷移ルール：

```
隣接セクション間の明度差: ±1段階以内
（bright → mid → dark は許容、bright → dark は禁止）

セクション背景色による自然な区切り（--bg / --bg-alt / --bg-dark）は
写真の明度変化とは独立して設定可能。
ただし写真がある場合、その写真の明度がセクション背景と著しく乖離しないこと。
```

## 9. 技術仕様

ヒーロー：1920×1080px以上。セクション：800×600px以上。ギャラリー：600×600px or 600×800px。OGP：1200×630px。圧縮：WebP 80%・200KB以下。loading="lazy"（ヒーロー除く）。width/height必須。alt属性：日本語・30文字以内。

### モバイルクロップ確認（必須）

PCで選んだ写真がモバイル（375px幅）で主役が切れていないか確認すること。

| 写真タイプ | モバイル時の注意 |
|-----------|----------------|
| 人物（横位置） | 顔が中央寄りでないと左右がクロップされて頭が切れる |
| 建物（縦位置） | 上下クロップで空や基礎が消える |
| 料理（引き） | 皿が小さく見えすぎる → モバイルはクローズアップを別途用意 |
| Hero全幅 | `object-position: center top` 等でフォーカスを指定 |

CSS対応例：
```css
.hero-img { object-fit: cover; object-position: center 30%; }
@media (max-width: 640px) { .hero-img { object-position: center 20%; } }
```
